# Real Estate Management System

A desktop application for managing real estate records, built with Python, Tkinter, SQLite, and Object-Oriented Programming.

## About the Project

This project was developed for a real contractor who needed a simple desktop application for managing real estate-related records.

The original data was stored in a legacy Visual FoxPro 9 database. I exported the required data and designed this application based on the contractor's workflow, needs, and preferred graphical interface.

This is also my first object-oriented programming project. The goal was to practice OOP concepts such as classes, objects, encapsulation, and separation of responsibilities while building a practical application for a real use case.

## Features

- Manage real estate records
- Store data locally using SQLite
- Automatically create the database file if it does not exist
- Simple desktop graphical interface
- Object-oriented code structure
- Built based on a real user's requirements

## Technologies

- Python
- Tkinter
- SQLite
- Object-Oriented Programming

## Requirements

Before running the application, make sure the following requirements are met:

- Python 3.12+
- Tkinter
- SQLite
- Arial font

The application uses a Persian graphical user interface. For the intended appearance, make sure the Arial font is available on your system.

## Background

The contractor previously used data stored in Visual FoxPro 9. Since the old system was not suitable for the new workflow and desired interface, this application was created as a modern and simpler replacement using Python.
